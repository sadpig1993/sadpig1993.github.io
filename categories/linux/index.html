CentOS挂载NTFS硬盘

CentOS本身是不支持NTFS的格式的，为了让CentOS能够支持Windows的NTFS。利用ntfs－3g使其支持NTFS的挂载。

步骤如下：

一、安装ntfs－3g

1、从官网http://www.tuxera.com/community/ntfs-3g-download/下载稳定版ntfs－3g

2、解压tar.gz包。在目录中运行命令窗。







  1. ./configure

  2. make

  3. make install



或者直接都在命令窗中操作










  1. wget http://tuxera.com/opensource/ntfs-3g_ntfsprogs-2011.4.12.tgz

  2. tar zxvf ntfs-3g-2010.8.8.tgz

  3. cd ntfs-3g-2010.8.8

  4. ./configure

  5. make

  6. make install



*在configure的时候可能会跳出 no acceptable C compiler in $PATH



这是由于没有安装gcc这个c编译器，安装方法 # yum -y install gcc

重新上面configure后的步骤

*解决*

一、挂载NTFS硬盘

这里以windows中的C盘( /dev/sda1)为例。

1、创建挂载点，建议放在/mnt/windows下






  1. mkdir /mnt/windows

  2. mkdir /mnt/windows/c


2、挂载C盘




  1. mount -t ntfs-3g /dev/sda1 /mnt/windows/winC

  2. mount -t ntfs-3g /dev/sda5 /mnt/windows/winD

  3. mount -t ntfs-3g /dev/sda6 /mnt/windows/winE

  4. mount -t ntfs-3g /dev/sda7 /mnt/windows/winF

  5. mount -t ntfs-3g /dev/sdb1 /mnt/windows/MobileHard


至此，就可以查看并编辑/mnt/window/中Windows中NTFS盘的内容

nginx代理proxy_pass设置

今天在服务器上装了node和express

用forever守护进程自启动

express用的8080端口，然后用nginx反向代理把域名代理到了127.0.0.1:8080了。

server
{
listen 80;
server_name node.miyijia.com;
location / {
proxy_pass http://127.0.0.1:8080;
proxy_redirect off;
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
}
